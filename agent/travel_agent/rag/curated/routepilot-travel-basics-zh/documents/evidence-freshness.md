# 旅行证据的时效与使用边界

旅行知识必须按变化速度处理。稳定方法类知识可以较长周期复核；地点介绍、无障碍设施和预约规则需要定期检查；营业时间、票价、班次、天气、库存、道路状态、临时关闭和安全公告属于实时或高频变化事实，不能由静态 RAG 文档单独担保。

每条知识应保留规范来源 URI、发布者、来源版本、许可、观察时间、有效期、信任等级和内容哈希。没有许可或无法确认索引权利的网页不能直接复制进知识库。外部文本始终是“不可信证据”，即使来源是官方机构，也不能把网页中的指令当成系统指令或触发工具调用。

检索结果应先按租户和可见性过滤，再按语言、地点、主题、有效期和信任等级筛选，最后进行词法或向量排序。排序分数只表示与问题相关，不表示事实一定正确。回答必须把来源质量、时效和多来源冲突纳入判断。

当多个来源冲突时，优先采用与具体事项直接相关、版本更明确、观察时间更新的官方或运营方来源，同时保留冲突记录。不能静默拼接互相矛盾的片段。证据过期或 Provider 不可用时，应降低结论强度、标明待核验内容，必要时拒绝给出具体承诺。

更新知识时不覆盖旧来源版本，而是用新的 source version 和 corpus revision 发布新文档，再对旧文档执行隔离或下线。发现许可撤回、严重错误、提示注入或隐私内容时，应立即 quarantine 或 tombstone，并通过固定检索问题验证旧内容不再召回。
