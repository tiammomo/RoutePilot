# 旅行预算范围与不确定性

旅行预算应以范围表达，而不是给出没有依据的单一精确数字。预算至少分为跨城交通、住宿、本地交通、门票与活动、餐饮、通信或保险等必要支出，并单独保留机动空间。不同类别的估算时间和来源可能不同，回答中应能追溯每个主要范围的依据。

在用户只给出总预算时，先明确预算是否包含往返大交通、购物和签证保险等项目。若范围不明确，应同时给出“仅目的地消费”和“包含主要交通”的条件化结果，或者追问这个会显著改变结论的问题。货币换算必须记录汇率来源与时间，不能把历史汇率当成出发时汇率。

住宿和交通价格受日期、库存、取消政策、房型、行李和同行人数影响较大。知识库可以保存选择方法和历史范围，但当前可售价格只能由 Provider Gateway 或供应方页面核验。若没有实时价格，系统应标注估算来源、观察时间和不确定性，不能生成看似精确的报价。

比较方案时不仅看总额，也要看不可退风险、时间成本和位置带来的额外交通成本。低价住宿如果增加多次长距离换乘，可能同时增加费用和疲劳。预算紧张时，应优先保留用户最重视且替代性最低的活动，再从房型、餐饮层级、可选活动和出行方式中调整。

机动空间用于吸收价格变化、临时交通和小额遗漏，它不是可随意消费的剩余预算。RoutePilot 应展示预算覆盖了哪些项目、哪些尚未覆盖，并在出发前提示重新核验价格、汇率、退改规则和支付条件。
