# 行程节奏与空间组织方法

可执行行程应先按地理邻近和交通连接组织活动，再安排景点数量。每天围绕一个主要区域或一条自然动线展开，通常比在城市两端反复往返更稳定。跨区移动应被视为一个有时间成本和失败概率的活动，而不是行程表中的空白。

每天要区分核心活动、可选活动和恢复时间。核心活动是即使发生延误也希望保留的内容；可选活动可以在天气、体力或交通变化时删除；恢复时间包括用餐、休息、找路、排队和临时调整。系统不应把所有可用时间填满，也不应默认交通、入场和离场能够无缝衔接。

节奏判断需要结合同行人。带老人、儿童、轮椅使用者或容易疲劳的同行者时，应减少连续步行和频繁换乘，优先安排可坐下休息、可中途退出且替代方案明确的活动。是否真正无障碍必须依据具体场所和交通运营方的最新信息核验，不能从“有电梯”或“无障碍入口”单一字段推断整条路线可用。

抵达日和离开日应降低安排密度，并为行李寄存、入住规则、机场或车站通勤保留余量。需要预约或有固定入场时间的活动先锁定，其他活动再围绕它排列。连续多日旅行应避免每天都设置相同强度，可以在高强度日之后安排更灵活的半日或室内活动。

行程中的时间建议属于规划估算，不等于实时承诺。路线时间、排队、天气和临时关闭在出发前仍需通过 Provider Gateway 或官方来源确认。系统应保留计划版本，让用户在约束变化后重新生成，而不是覆盖原方案。
