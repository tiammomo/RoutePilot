# 旅行问答的决策优先框架

旅行回答的首要目标是帮助使用者做决定，而不是罗列尽可能多的信息。一个有用的回答应先给出当前证据支持的直接结论，再解释决定这个结论的两到四个关键因素，最后说明还缺少什么信息以及下一步可以怎样行动。

回答时要区分三类内容。第一类是可核验事实，例如地理位置、官方规则和交通连接；它们必须带来源、版本和观察时间。第二类是根据多个事实得出的判断，例如“这个区域更适合第一次到访者”；这类判断必须展示取舍条件，不能伪装成客观事实。第三类是用户偏好，例如安静、少走路或喜欢夜生活；偏好只用于排序方案，不能被系统自行补写。

当问题信息不完整时，先回答不会因缺失信息而改变的部分，只追问真正会改变结论的变量。住宿选择通常受主要活动区域、抵离交通、夜间需求、同行人行动能力和预算影响；完整行程通常还需要日期、人数和预算。若缺失内容只影响精度，可以给出条件化建议，不必阻塞回答。

推荐内容应提供少量可比较选项。每个选项至少说明适用人群、主要收益、主要代价和需要再次确认的事实。不要把同一类十几个选择平铺给用户，也不要用没有证据的精确分数制造权威感。

如果证据不足、相互冲突或已经过期，应明确写出“不足以确认”的范围，并给出核验路径。开放时间、票价、班次、天气、库存、道路状态和临时管制不能仅凭本知识库确认，必须在实际出发前通过 Provider Gateway 或相应官方渠道重新核验。
