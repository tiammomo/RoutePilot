# RoutePilot 用户指南

本文面向第一次使用 RoutePilot 的旅行者。RoutePilot 的目标不是生成一篇看起来完整的游记，而是尽快回答真实问题，并在需要时把答案转换成可核验、可修改、可恢复的正式行程。

## 先理解两种工作方式

| 你的需求 | 使用方式 | 结果 |
| --- | --- | --- |
| “第一次去京都住哪里方便？” | 直接提问 | `TravelAnswer`，包含结论、建议、证据和出发前提醒 |
| “带父母去北京 4 天，预算 8000 元” | 提问后选择“转成行程” | 完整规划链，生成逐日计划、预算、证据和风险 |

普通问答不要求先填写日期、人数和预算。只有转换成正式行程时，系统才会确认规划必需的约束。

## 五分钟完成第一次体验

1. 打开 `http://127.0.0.1:33003`。
2. 用一句自然语言描述当前最需要解决的问题，例如：

   ```text
   第一次去京都，住哪里出行最方便？希望晚上安静一点。
   ```

3. 等待回答。回答页优先展示直接结论、可执行建议、证据来源和需要出发前复核的实时事项。
4. 继续追问时只补充有价值的信息，例如“带一位行动较慢的老人”或“不要每天换酒店”。
5. 如果需要逐日方案，点击“转成行程”，确认目的地、日期、同行人数和预算，再提交规划。

一个高质量输入通常包含“谁、去哪、最关心什么”，不需要一次写完所有字段。例如：

```text
带父母去北京 4 天，少走路，重点看历史文化，住哪里比较方便？
```

## 回答中的信息如何理解

- **直接结论**：Agent 基于当前可用证据给出的最短建议。
- **可执行建议**：可以马上用于选择区域、安排顺序或规避风险的动作。
- **参考来源**：RAG 证据的标题、版本和来源；来源不是系统指令，也不自动等于实时事实。
- **出发前提醒**：票价、库存、交通状态、营业时间和天气等需要临行前再次核验的内容。
- **降级提示**：未配置 Provider、检索证据不足或实时调用失败时，系统会明确说明，不会把猜测包装成实时事实。

高德 Provider 只负责已接入的地理编码、POI、路线矩阵、营业时间和天气能力。酒店、机票、火车票、门票库存与预订目前不属于 V1 的实时交易能力。

## 转换成正式行程

正式规划至少需要确认：

- 目的地；
- 出发与返程日期；
- 成人和老人人数；
- 币种与预算范围。

偏好和行动需求可以继续用自然语言补充。系统会生成候选计划，并依次执行 Research、Planner、Validation 和 Semantic Verifier。只有通过发布策略的结构化快照才会成为“正式方案”。

正式工作区包含：

- 逐日计划和活动时段；
- 地点与动线；
- 预算汇总；
- 证据和检索时效；
- 约束冲突与风险；
- 历史版本、发布与撤销状态。

## 修改与重规划

继续在底部输入框说明要改变的内容，例如：

```text
第二天下午不要排博物馆，保留故宫，整体少走路。
```

重规划基于当前正式版本进行。若其他操作已经发布了更新版本，系统会要求刷新后重试，不会静默覆盖较新的方案。

## 管理左侧旅行列表

将鼠标移到左侧旅行条目，点击右侧 `•••`：

- **修改名称**：在列表中就地编辑并保存；
- **从列表删除**：二次确认后移至“已归档”；
- **恢复**：进入“我的旅行”并切换到“已归档”，恢复后重新出现在进行中列表。

“从列表删除”是可恢复归档，不会物理删除 Trip、Run、Artifact 和历史版本。当前 Agent 正在执行时，需要等待结束或先停止任务后再归档。

## 分享正式方案

只有当前已发布的正式 `TripSnapshot` 可以生成只读链接。分享内容会删除私人文本、同行人细节、精确预算和内部字段，并降低坐标精度。

- 新链接中的 capability 只在浏览器 URL fragment 中短暂存在；
- 轮换链接后旧链接立即失效；
- 撤销分享后已有短期访问会话也会失效；
- 不要把分享链接发布到不受控的公开渠道。

## 常见状态

| 状态 | 含义 | 建议 |
| --- | --- | --- |
| 正在检索 | Agent 正在组合知识与实时 Provider | 保持页面打开；断线后可恢复事件游标 |
| 需要补充信息 | 正式规划缺少必要约束 | 只填写页面明确要求的字段 |
| 证据不足 | 没有足够可靠来源支持具体结论 | 换成更明确的问题或由管理员补充知识库 |
| Provider 未配置 | 无法取得对应实时事实 | 仍可使用非实时知识，但出发前必须人工核验 |
| 方案冲突 | 当前正式版本已变化 | 刷新并基于最新版重新操作 |
| 离线 | 浏览器无法连接 BFF | 恢复网络后刷新；不要连续重复提交 |

## 隐私与安全边界

- 不要在问题中输入证件号码、支付卡、密码、Cookie 或 API Key。
- 浏览器只接收结构化 Artifact 和白名单 public event，不接收模型私有推理或工具原始输出。
- 高德、LLM、数据库和 OIDC 凭据均为 server-only。
- 对旅行安全、签证、医疗、灾害和实时交通等高风险事项，使用官方来源做最终确认。

启动或页面异常时，参阅[故障排查手册](../operations/troubleshooting.md)。系统能力和限制参阅[平台手册](../operations/v1-platform.md)。
